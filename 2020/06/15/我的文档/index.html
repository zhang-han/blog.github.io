git初始化和提交流程

  • git pull
  • git push

git管理地址和初次推送流程

  • git remote add origin urllll
  • git push –set-upstream origin master

git排斥不想提交的文件

  • git rm -rf –cache xxx.iml

git关联远程module仓库

git submodule add urllll filename/
or
git submodule update –init
git submodule sync

git查看远程仓库地址

  • git remote -v

    本地所有修改的。没有的提交的，都返回到原来的状态

    git checkout .

    把所有没有提交的修改暂存到stash里面。可用git stash pop回复。

    git stash

    返回到某个节点，不保留修改。

    git reset –hard HASH

    返回到某个节点。保留修改

    git reset –soft HASH

github与hexo使用blog

  • github新建仓库blog
  • blog仓库setting，在github pages中选择风格，完成后setting中就有github pages了
  • 本地文件找个地方新建文件夹blog
  • 去hexo.io中执行命令
    • npm install hexo-cli -g
    • hexo init blog
    • cd blog
    • npm install
    • hexo server

blog使用

  • hexo g 编译
  • hexo d 部署
  • hexo s 启服务
